# 远方智设标题设计引擎 v1

## 1. 当前阶段判断

远方智设已经完成了背景主视觉生成、designFamily、layoutFamily、displayPolicy、showMascot、font-library、title-art-styles 和 Sharp 后期文字合成。现在系统已经不是单纯的图片生成实验，而是逐步走向品牌受控的设计生产工具。

但当前成品仍然有一个明显短板：标题层还停留在“字体渲染 + 描边 + 阴影”的阶段，离历史设计稿里那种“标题被设计过”的感觉还有距离。

当前阶段可以粗略判断为：

- 背景主视觉：70-75 分
- 版式系统：50-60 分
- 标题字体设计：30-40 分
- 整体成品：55-60 分

短板不在 AI 背景，也不在是否能叠字，而在标题层。远方智设不能只是“背景图 + 普通文字合成”，它应该是“背景主视觉 + 标题设计系统 + 品牌信息合成”。

## 2. 核心原则

- 不让 AI 直接生成中文标题图片。中文标题一旦交给图像模型直接生成，会有错字、乱码、笔画错误、字形崩坏和品牌不可控风险。
- 主标题和副标题必须由 Sharp 后期稳定合成，确保文字内容准确。
- AI 可以作为“字体导演 / 标题导演”，但只能输出结构化 JSON，不能直接画中文字。
- 字体库必须可控，只能从系统允许的 font-library 中选择。
- 字体文件不提交 GitHub，也不作为下载资源提供给用户。
- 用户不需要选择复杂字体参数。系统应根据 designFamily、layoutFamily、displayPolicy 和活动内容自动选择标题设计。
- 免费字体只是原材料，真正重要的是标题设计系统。
- 标题不是“选字体”，而是“设计标题视觉资产”。

标题设计系统至少包括：

- 字体骨架
- 描边 / 阴影 / 光效
- 字形节奏
- 主题装饰
- 标题版式位置
- 背景可读性适配

## 3. 标题设计引擎分层

### 3.1 字体骨架

font-library 提供可控、可授权、可替换的字体资产。每个字体都需要记录用途、限制、适合场景和不适合场景。

字体骨架决定标题的气质底座：黑体偏商务和清晰，宋体偏文化和书卷气，圆体偏亲和，展示字体偏活动感。系统不能把字体当装饰随机使用。

### 3.2 标题效果

标题效果用于提高识别度和画面融合度，包括：

- 描边
- 阴影
- 光晕
- 渐变
- 高光
- 立体感

效果必须服务可读性和品牌感，不能为了“炫”牺牲中文准确性。招生、汇报、发布会可以更强；文学、国风、品牌理念应更克制。

### 3.3 字形节奏

标题设计不能只是一整行等宽排布。字形节奏需要支持：

- 大小错落
- 字间距
- 关键词强化
- 主副标题层级
- 局部倾斜
- 重点字放大

例如“暑假文学表达体验课”中，“暑假”“文学表达”“体验课”可以形成不同层级，而不是被当成一串普通网页标题。

### 3.4 主题装饰

主题装饰必须从活动语境中来，不是每张图都堆同一套符号。

- 国风：印章、云纹、金线、题签
- 阅读：书签、纸张、书页、铅笔线
- 成长：箭头、阶梯、光束、奖章
- 汇报：舞台光、麦克风、奖杯、作品卡片
- 招生：标签、按钮感、强色块

装饰应围绕标题服务，不能抢走标题，也不能变成低幼贴纸。

### 3.5 版式融合

标题位置不能固定。标题应和 layoutFamily、背景构图、displayPolicy 协作。

- stageGlow 可在舞台中心或中上区域形成仪式感
- modernChinese 可像题字横幅或现代国风题签
- literaryEditorial 可像杂志封面标题
- boldCampaign 可像强营销标题和活动标签

标题设计引擎要让标题成为画面主资产，而不是贴在背景上的普通文本。

## 4. 标题设计家族 v1

### 4.1 stageHeroTitle

适合：

- 成长汇报课
- 成果展示
- 发布会
- 教学比赛

视觉：

- 厚重黑体
- 实心蓝色标题
- 轻白描边
- 舞台光阴影
- 奖章 / 星光 / 聚光辅助

目标：

正式、有仪式感、有成果展示感。

### 4.2 modernChineseTitle

适合：

- 国学
- 诗词
- 名著
- 传统文化

视觉：

- 宋体 / 楷体骨架
- 蓝色或深色主标题
- 米白描边
- 金线 / 红印章 / 云纹 / 题签
- 不做廉价毛笔字

目标：

现代国风、文化感、教育品牌审美。

### 4.3 campaignTitle

适合：

- 招生
- 开班提醒
- 报名提醒
- 短期活动

视觉：

- 粗黑体 / 得意黑
- 高对比
- 白描边
- 强阴影
- 色块标签
- 关键词放大

目标：

快速抓住注意力，但避免廉价促销感。

### 4.4 literaryMagazineTitle

适合：

- 一本好书
- 读书节
- 文学周
- 品牌理念

视觉：

- 文楷 / 宋体
- 大字距
- 轻阴影
- 书签 / 纸张 / 杂志式排版
- 克制、有书卷气

目标：

文学感、编辑感、品牌气质。

### 4.5 ipEventTitle

适合：

- 暑期营
- 游园会
- 轻活动
- 品牌 IP 活动

视觉：

- 漫黑 / 圆体
- 活泼描边
- 小图标装饰
- 动势
- 但不低幼

目标：

活动感、趣味、品牌化。

### 4.6 cleanBrandTitle

适合：

- 通用通知
- 课程说明
- 家长群
- 正式说明图

视觉：

- 稳定黑体
- 高可读
- 克制阴影
- 少装饰
- 信息优先

目标：

清楚、可信、稳定。

## 5. 当前 Sharp 可实现范围

当前阶段可以先由 Sharp 实现：

- 字体选择
- 字号
- 字距
- 填充色
- 描边
- 阴影
- 光晕
- 多行排版
- 主副标题层级
- 标题位置随 layoutFamily 调整

暂时不要做：

- 真正复杂字形变形
- 手写生成
- AI 直接画字
- 任意字体上传
- 复杂曲线文字
- 过度艺术化导致可读性下降

v1 的目标不是一次做出所有标题效果，而是把标题从“普通文字层”推进到“可配置、可扩展、可验证的设计层”。

## 6. 后续 AI 字体导演

未来 AI 可以做字体导演，但只能输出结构化 JSON，不直接画字。

示例：

```json
{
  "titleDesignFamily": "stageHeroTitle",
  "titleArtStyle": "stageGlow",
  "fontKey": "sourceHanSansBold",
  "emphasisMode": "mainTitleSolid",
  "decorations": ["stageLight", "smallStars"],
  "stroke": {
    "color": "#FFFFFF",
    "width": 1
  },
  "shadow": {
    "color": "#0B1F3A",
    "dx": 0,
    "dy": 4,
    "blur": 5,
    "opacity": 0.38
  },
  "reason": "成长汇报课需要舞台和成果展示感，标题应实心稳重。"
}
```

约束：

- 只能从允许的 fontKey 里选
- 只能从允许的 titleDesignFamily 里选
- 颜色必须在远方 VI 和安全辅助色内
- 不允许改标题文字内容
- 不允许直接生成中文标题图像

AI 的价值是理解活动目标和视觉语境，然后选择标题设计参数；Sharp 的价值是稳定、准确、可重复地渲染真实中文文字。

## 7. 标题系统下一步施工顺序

第一步：优化现有 stageGlow，让标题更实心、更稳，不要空心描边感。

第二步：扩展 title-art-styles.ts 为 titleDesignFamily v2，增加 decorations / emphasisMode / layoutIntent。

第三步：在 image-compose.service.ts 中实现标题双层绘制：

- 先绘制描边/阴影层
- 再绘制实心文字层

避免 CSS paint-order 造成空心感。

第四步：实现 2-3 个装饰型标题：

- stageHeroTitle
- modernChineseTitle
- campaignTitle

第五步：做背景可读性检测：

- 标题区域亮度
- 背景复杂度
- 对比度
- 自动加描边/阴影/光晕

第六步：再考虑 AI 字体导演 JSON。

## 8. 禁止路线

- 不使用 AI 直接生成中文标题图片
- 不让用户上传任意字体
- 不接来源不明的免费字体合集
- 不把字体文件提交 GitHub
- 不把字体作为下载资源提供给用户
- 不让老师手动选复杂字体参数
- 不为了艺术效果牺牲中文准确性和可读性

## 9. 产品目标

远方智设要达到：

- 背景有主题差异
- 标题有设计感
- 中文文字准确
- 品牌 VI 统一
- 老师无需懂设计
- 输出可直接发朋友圈、家长群、小红书、公众号、活动现场物料
- 用一两千元/月的软件成本替代校区重复设计工作

最终状态不是“老师会调字体”，而是老师只填写设计需求，系统自动生成一张品牌统一、标题有设计感、信息准确、可以直接传播的成品图。
