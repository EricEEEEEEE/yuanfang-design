# 远方智设标准模式 Prompt 设计语言 v2

## 1. 当前问题复盘

标准模式从早期的“儿童课堂插画”逐步纠偏到“教育品牌活动主视觉背景”，方向是正确的，但最近测试暴露出新的问题：Prompt 过于防守，AI 被限制成只能生成安全、干净、可叠字的背景底板，导致不同主题之间差异不足。

### 1.1 过度禁止导致同质化

当前 Prompt 里有大量“不要生成”的约束。硬边界是必要的，尤其是文字、Logo、二维码、电话、校区名等内容必须禁止。但当“人物、儿童、老师、课堂、故事感、场景”也被强烈压制后，AI 能使用的视觉变量只剩书页、曲线、渐变、光效、山水等少数元素。

结果是：图变稳了，但也变像同一套底板。

### 1.2 visualBrief 权重太弱

`visualBrief` 已经进入 prompt，但目前只是一个较短的参考区块。它告诉 AI “这次主题是什么”，但没有进一步要求 AI 把这些词翻译成画面结构、主题符号、空间关系和视觉焦点。

结果是：AI 看到“四大名著、少年国学、古典文学启蒙”，但可能仍然只生成通用书页、山水、光效。

### 1.3 从儿童插画纠偏到主视觉后，又变成模板底板

之前的问题是“太像儿童绘本课堂插画”。纠偏后，我们强调“不画人物、不画课堂故事、不画孩子围坐”，这让画面更接近活动 KV，但也把主题叙事能力一并压低。

远方历史主视觉并不是完全无叙事的空背景。它通常有明确主题符号、较强排版预留、少量活动意象和品牌结构感。

### 1.4 AI 创意空间不足

目前 Prompt 主要告诉 AI：

- 不能生成什么
- 用哪些品牌色
- 要留干净区域
- 要像主视觉背景

但没有足够告诉 AI：

- 这张图的活动目标是什么
- 主题应该如何视觉化
- 哪些元素可以成为本次独特记忆点
- 构图如何和后期标题协作
- 同一主题下如何避免重复

v2 的核心目标是：保留硬边界，放松软约束，增强 creative brief 和 visual translation。

## 2. 新 Prompt 原则

### 2.1 output goal

AI 输出不是完整海报，也不是故事插画，而是“可用于后期合成的活动主视觉背景”。

目标应明确为：

- 竖版 2:3 活动主视觉背景
- 可承载强标题排版
- 有主题记忆点
- 有远方品牌气质
- 后期可叠加 Logo、吉祥物、标题、校区信息和二维码

### 2.2 brand DNA

品牌气质不只等于品牌色。远方标准模式应持续体现：

- 儿童文学教育
- 阅读与表达
- 成长与打开世界
- 服务于考试且高于考试
- 明亮、积极、有教育品质感
- 少年儿童适用，但不低幼
- 有文学想象力，但不散漫

### 2.3 creative brief

`visualBrief` 应成为本次生成的主题驱动器，而不是弱参考词。

Prompt 应要求 AI：

- 阅读 visualBrief
- 提炼 2-4 个核心意象
- 将核心意象转化为背景结构
- 让画面拥有本次活动的独特识别点
- 不把 visualBrief 里的文字直接写进画面

### 2.4 visual translation

主题词不能停留在文字层，必须转译成视觉元素。

示例：

- “四大名著”：古典书页、章回小说卷轴、山水层次、人物剪影式符号，但不生成可读文字
- “飞花令”：飞散花瓣、诗词卷轴、月光、节奏感光轨，但不生成文字
- “春季招生”：生长路径、打开的书、向上的光、课程升级感
- “文学周”：书页空间、活动舞台感、纸张层次、文学世界展开

### 2.5 composition strategy

构图要先服务后期合成，再服务主题表达。

建议结构：

- 主题视觉元素占画面 25%-45%
- 预留 35%-45% 低复杂度排版区域
- 不要让留白变成空白模板
- 主视觉焦点可以在中部、下部或斜向动线，不固定角落
- 辅助元素可延展到边缘，但不能包围成彩带边框

### 2.6 style dial

风格不应只写“温暖、活泼、国风、文艺”，而应给出可调节的视觉拨杆：

- 温暖：可信、亲切、课程发布感、家长传播友好
- 活泼：明亮、有秩序、有品牌活动感，不低幼、不贴纸
- 国风：现代国风 KV，诗词、名著、卷轴、山水、光效，不古板
- 文艺：书页空间、窗光、纸张层次、文学想象，不小清新手账

### 2.7 variation rule

每次生成都应基于 visualBrief 产生差异，而不是只替换主题标签。

建议规则：

- 至少选择 2 个 visualBrief 相关意象作为主视觉记忆点
- 不同主题不要复用同一套“书页加渐变加曲线”结构
- 同一风格下也要改变视觉焦点、空间层次和主题符号
- 如果主题是国学、诗词、名著，允许更强的文化符号
- 如果主题是招生、试听课，允许更强的课程发布和成长路径结构

### 2.8 hard boundaries

硬边界仍然必须清晰、稳定、重复出现。它们是品牌资产和后期合成的安全底线。

### 2.9 post-production reserved areas

AI 背景必须为 Sharp 合成预留区域，但不能把画面做成空模板。

要求：

- 为主标题、副标题、校区信息、电话、二维码、Logo、吉祥物预留低复杂度区域
- 不固定所有预留区域到同一个角
- 低复杂度区域应有层次、光感或浅色块，不是纯空白
- 主题视觉不能压住标题区域
- Logo 和吉祥物由后期叠加，AI 不生成

### 2.10 与标题系统和渲染层的边界

标准模式背景不是完整海报，也不是标题字体渲染层的下游。正确关系是：

用户输入 → 品牌配置横向约束 → 图片 / 背景 / 空间策略层 → 背景空间分析与 spatial strategy → 标题结构设计层 → Vector Glyph Renderer / Sharp Raster Measurement → Final Composer。

原则：

- 图片 / 背景 / 空间设计层在标题字体渲染层上游。
- 字体根据图片与空间策略走，不是图片根据字体走。
- 背景 prompt 应服务 background layout、negative space、safe zones 和后期标题资产进入，而不是服务某个固定字体模板。
- 品牌配置是横向约束，约束图片生成、标题结构、字体选择、颜色、Logo、吉祥物和最终合成。
- Vector Glyph Renderer 是下游标题资产渲染器，不是驱动 AI 背景生成的上游。
- Final Composer 负责把背景、标题资产、Logo、吉祥物、校区信息合成最终海报。
- 不允许把项目走成“先做一套漂亮字体，再硬塞到图上”的模板海报路线。

## 3. 硬边界

以下规则必须保留，不应被 visualBrief、主题、风格或元素覆盖：

- 不生成任何文字
- 不生成中文标题
- 不生成英文标题
- 不生成假汉字
- 不生成乱码字
- 不生成 Logo
- 不生成远方 Logo
- 不生成类似远方 Logo 的花瓣标志
- 不生成官方大象吉祥物
- 不生成二维码
- 不生成电话
- 不生成校区名称
- 不生成地址
- 不生成可读品牌标语

## 4. 软约束

以下内容不应继续作为绝对禁止项，而应改成“受控允许”。

### 4.1 人物

允许出现小比例、非真实照片感、非主体化的人物剪影或插画人物，用于表达活动氛围。

限制：

- 不要真实儿童照片感
- 不要人脸特写
- 不要人物占据画面主体
- 不要多人挤满课堂

### 4.2 儿童

允许出现小学阶段儿童的背影、剪影、侧影或小比例活动姿态。

限制：

- 不要幼儿园感
- 不要低幼绘本风
- 不要真实儿童肖像
- 不要以儿童表情作为画面卖点

### 4.3 老师

允许出现老师的抽象引导姿态、剪影或局部场景符号。

限制：

- 不要老师讲台授课成为完整故事
- 不要传统补习班广告感
- 不要真实照片感

### 4.4 故事感

允许轻微故事感，用来让主题更有记忆点。

限制：

- 不要完整绘本故事插画
- 不要复杂剧情
- 不要让故事压过活动主视觉结构

### 4.5 场景

允许课堂、书房、活动舞台、阅读空间、山水空间等场景化背景。

限制：

- 场景应服务主视觉
- 不要画满细节
- 不要像完整课堂照片
- 不要变成通用教育插画

### 4.6 主题符号

应鼓励主题符号进入画面，这是解决同质化的关键。

可控方向：

- 书页、卷轴、山水、月亮、竹子、亭台、纸张、窗光
- 舞台光效、作品墙、成长路径、打开世界、文学人物剪影
- 花瓣、飞鸟、云纹、星光、纸飞机等可少量使用

限制：

- 不要生成可读文字
- 不要生成 Logo 化符号
- 不要符号堆砌

## 6. JSON 模板改造建议

本轮只新增文档，不修改模板。后续建议按以下方向改造。

### 6.1 `templates/_base.json`

建议调整：

- 把“默认不生成人物、不生成儿童、不生成老师”改为“人物受控允许”
- 增加 output goal
- 增加 composition strategy
- 增加 variation rule
- 保留 hard boundaries
- 强化“背景主视觉不是纯底板”

### 6.2 `templates/_brand-rules.json`

建议调整：

- 保留品牌色、Logo 规则、吉祥物规则
- 增加 brand DNA 描述
- 将 allowedVisualMotifs 区分为品牌母题、文学母题、活动母题

### 6.3 `templates/standard/themes.json`

建议调整：

- 每个主题从“限制型描述”改为“活动目标 + 视觉转译”

- 增加每个主题的 avoidSameness 提醒

### 6.4 `templates/standard/styles.json`

- 每个风格加入 style dial

### 6.5 `templates/standard/elements.json`

- 例如 books 可以是“书页空间结构”，不是简单几本书

### 6.6 `src/services/template.service.ts`

- 将 `visualBrief` 区块提前到主题、风格、元素之前

- 增加 Visual Translation 和 Variation Rule 区块
- 继续保证用户文案字段不进入 prompt
- 继续保证 overlayData 只返回 Sharp 合成字段

## 7. 前端输入形态：从选项选择器升级为引导式创意简报

当前标准模式前端主要让用户选择主题、风格、元素，再填写标题和校区信息。这种输入方式稳定、易用，但开放性不足。对 AI 来说，`theme / style / element` 更像模板分类标签，而不是完整设计需求；当用户真实活动信息没有充分进入 prompt 时，不同主题很容易生成相似的背景底板。

远方智设不能只是模板选择器。它更应该像一个设计师，在生成前引导校区老师把“这次活动是什么、希望画面传达什么、哪些元素要出现、哪些内容不要出现”说清楚。用户不应该感觉自己在写 Prompt，而应该感觉自己在填写一份清晰、友好的设计需求。

推荐采用“结构化引导 + 自由描述”的输入方式：关键字段用简短问题引导，保留少量选择项降低使用门槛，同时给用户足够空间描述真实活动。

### 7.1 建议字段

- `productOutputType`：本次要生成什么物料，例如朋友圈海报、家长群宣传图、公开课活动图、读书节主视觉、课程发布图。
- `eventBrief`：这次活动是什么，例如“四大名著少年国学公开课”“春季阅读表达招生”“飞花令诗词挑战”。
- `styleBrief`：希望画面是什么感觉，例如“现代国风、有文学感、不要太幼稚”“明亮、可信、有课程发布感”。
- `visualDetails`：希望画面中出现哪些元素，例如“卷轴、山水、打开的书、月亮、少年剪影”“书页空间、窗光、文学世界展开”。
- `avoidNotes`：有哪些内容不要出现，例如“不要低幼卡通、不要真实儿童照片、不要补习班广告感、不要人物太多”。
- `mainTitle`：主标题。
- `subtitle`：副标题。
- `campusName`：校区名称。
- `campusAddress`：校区地址。
- `campusPhone`：联系电话。

### 7.2 字段分工

`theme / style / element` 可以保留为辅助选项，用于快速建立基础方向和默认模板片段。但真正驱动画面差异的应该是：

- `eventBrief`
- `styleBrief`
- `visualDetails`
- `avoidNotes`

这些字段应进入 AI prompt，用于创意背景生成。它们不应被简单拼成一段弱参考，而应被包装成 Creative Brief、Visual Translation、Composition Strategy 和 Variation Rule。

`mainTitle / subtitle / campusName / campusAddress / campusPhone` 仍然只作为后期合成数据，由 Sharp 渲染到最终 JPEG。它们不应直接进入 AI prompt，也不应让 AI 生成文字。

### 7.3 前端体验原则

前端问题文案应像设计师在问需求，而不是像工程工具在收集 prompt 参数。

建议表达方式：

- “这次活动是什么？” 对应 `eventBrief`
- “希望画面给家长什么感觉？” 对应 `styleBrief`
- “你希望画面里有哪些元素？” 对应 `visualDetails`
- “有哪些内容不要出现？” 对应 `avoidNotes`
- “海报上要写什么？” 对应 `mainTitle / subtitle`
- “校区信息是什么？” 对应 campus 字段

这样用户填写的是设计需求，系统再负责把需求翻译成可控 prompt。

## 8. 大品牌设计语言借鉴层：方法论吸收，不是视觉模仿

远方智设不应该模仿任何大品牌的表面风格，也不应该把 Apple、Dyson、MUJI、Aesop、IKEA 的视觉外观直接套到教育海报上。真正值得吸收的是它们背后的设计方法：如何建立信息秩序、如何表达问题解决、如何使用克制留白、如何回应场景语境、如何规模化交付可用设计。

这一层的作用是给远方 Prompt 体系提供判断框架：生成图不是单纯“好看”，而是要帮助校区低成本、稳定地生产品牌受控、信息清晰、可直接传播的设计物料。

### 8.1 Apple：信息秩序与内容优先

#### 设计语言观察

Apple 值得借鉴的不是银灰色科技风，也不是冷淡极简感，而是清晰、克制、层级、内容优先、留白服务理解。Apple 的视觉系统通常让用户先看见核心信息，再看见产品与氛围；背景、光影、材质和空间都服务于信息被理解，而不是抢夺注意力。

#### 可吸收原则

远方可以吸收 Apple 的信息层级方法：家长必须一眼看懂活动是什么、适合谁、为什么值得关注。海报背景不能压过主标题和活动价值，视觉氛围必须服务“理解”，而不是只制造装饰感。

留白不是装饰，也不是高级感的固定套路。留白的价值在于让信息被看见，让主标题、课程价值和校区信息形成稳定层级。

#### 转译到远方智设的规则

- 标题区域必须清楚。
- 视觉元素不能抢走主标题。
- 背景复杂度要分区控制。
- 信息层级要稳定。
- 主题视觉负责记忆点，低复杂度区域负责承载标题和传播信息。

#### 不吸收什么

- 不做冷淡科技风。
- 不把远方变成 Apple 风。
- 不使用银灰、玻璃、金属质感作为远方默认风格。
- 不为了极简而削弱文学教育的温度和品牌色彩。

#### 可进入 prompt 的表达方式

```text
背景复杂度要分区控制：标题区低复杂度，主题视觉区有记忆点，辅助区保持克制；画面必须服务家长快速理解活动价值。
```

### 8.2 Dyson：问题解决与结构表达

#### 设计语言观察

Dyson 值得借鉴的是从问题出发、用结构和功能表达解决方案。它的设计语言不是“装饰一个产品”，而是让用户感知这个产品解决了什么问题、结构如何工作、价值在哪里。

#### 可吸收原则

远方每张图也要回答一个传播问题：招生图解决什么家长焦虑，汇报课展示什么教学成果，公开课为什么值得到校体验，课程发布为什么值得报名。设计不能只是漂亮，而要表达课程价值和传播目标。

#### 转译到远方智设的规则

- 招生图要表现成长路径、课程升级、家长信任。
- 汇报课要表现成果、表达力、荣誉感。
- 课程发布要有清楚的结构和视觉焦点。
- 可以使用路径、光束、模块、奖章、舞台、作品墙等结构化元素。
- 每个物料先明确传播目标，再选择视觉结构。

#### 不吸收什么

- 不做机械产品风。
- 不做冷硬科技风。
- 不把课程价值表达成机械结构或工业剖面。
- 不牺牲教育品牌的亲和力。

#### 可进入 prompt 的表达方式

```text
请先理解本次物料要解决的传播问题，再把它转化为清晰的视觉结构，例如成长路径、成果展示、课程升级、舞台聚光或模块化空间。
```

### 8.3 MUJI：克制留白与可承载空间

#### 设计语言观察

MUJI 值得借鉴的是空、克制、无过度装饰。它的空不是无内容，而是让物品、功能和使用者进入画面。克制不是乏味，而是减少噪音，让真正重要的东西被承载。

#### 可吸收原则

远方需要“有内容承载力的留白”，不是纯空白模板。复杂主题也要表达得干净，但远方不能变成无品牌的白板。留白应该有光感、纸张层次、浅色结构或空间感，让后期文字有落点，同时让主题仍然存在。

#### 转译到远方智设的规则

- 复杂主题也要表达得干净。
- 留白要有光感、纸张层次、浅色结构或空间感。
- 不要堆满符号。
- 不要为了高级而过度素淡。
- 少量关键符号优于大量装饰堆砌。

#### 不吸收什么

- 不做过度素淡。
- 不丢失远方色彩和活力。
- 不把背景做成无品牌、无记忆点的白板。
- 不把所有活动都处理成同一种安静生活方式画面。

#### 可进入 prompt 的表达方式

```text
预留区域应有温和层次、材质、光感或浅色结构，不是纯空白；主题元素要克制，用少量关键符号建立记忆点。
```

### 8.4 Aesop：场景语境与主题独特性

#### 设计语言观察

Aesop 值得借鉴的是每个空间回应自己的文化和场景，而不是复制同一模板。它保持品牌一致，但会根据城市、街区、材质、文化语境产生不同空间语言。

这正是远方解决同质化最重要的参考：品牌一致不等于每张图构图一样，不同活动必须有不同主题语汇。

#### 可吸收原则

远方的品牌一致应来自 VI、色彩、信息层级、后期合成规范和教育品牌气质，而不是所有图都复制同一种书页山水底板。每个活动都应该根据自己的语境生成专属视觉语汇。

#### 转译到远方智设的规则

- 国学可以用卷轴、山水、古典建筑。
- 招生应使用成长路径、课程价值、现代教育空间。
- 汇报课应使用作品墙、舞台光、荣誉感。
- 读书节应使用书页空间、阅读灯光、文学世界。
- 公司活动应使用商务发布会、数字纪念、品牌色块。

#### 不吸收什么

- 不做高冷奢侈品风。
- 不做家长看不懂的艺术装置感。
- 不为了独特而牺牲活动信息。
- 不把教育传播变成小众审美实验。

#### 可进入 prompt 的表达方式

```text
请根据本次活动语境生成专属视觉语汇。品牌一致来自远方 VI、色彩、层级和后期合成规范，而不是复制同一种书页山水底板。
```

### 8.5 IKEA：规模化可用与成本效率

#### 设计语言观察

IKEA 值得借鉴的是 form、function、quality、sustainability、low price 的平衡。它不是只追求漂亮，也不是只追求便宜，而是在大规模生产和真实使用之间找到可持续、可交付、可理解的设计方式。

#### 可吸收原则

远方智设的商业目标是替代校区设计师的一部分重复工作，让校区低成本稳定产出设计物料。生成图必须可直接使用、可批量生成、可复用，不是一次性艺术图。

#### 转译到远方智设的规则

- 老师可以快速填写需求。
- 输出要能直接发朋友圈、家长群、公众号、线下物料。
- 不依赖设计师二次加工。
- 美观、信息、效率、品牌一致要同时成立。
- 设计系统应支持批量生产，而不是每次重新发明一张图。

#### 不吸收什么

- 不做家居零售风。
- 不把设计降级成廉价模板。
- 不牺牲品牌质感换取低成本。
- 不让校区老师承担复杂设计判断。

#### 可进入 prompt 的表达方式

```text
生成结果必须可直接用于校区传播，不是实验性艺术图；设计应兼顾美观、信息清晰、品牌一致、可批量复用和低维护成本。
```

### 8.6 远方融合原则

远方智设不是模仿任何品牌，而是吸收：

- Apple 的信息秩序
- Dyson 的功能目标
- MUJI 的克制留白
- Aesop 的主题语境
- IKEA 的规模化可用

融合成远方自己的设计语言：

1. 信息先于装饰
2. 目标先于风格
3. 克制不等于空
4. 一题一语汇
5. 品牌一致不等于模板一致
6. 商务教育感优先
7. 可批量落地
8. 创意开放，品牌收口
